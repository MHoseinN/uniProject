# سیستم مدیریت پروژه‌های دانشجویی

سیستم جامع مدیریت پروژه‌های دانشجویی از مرحله انتخاب موضوع تا دفاعیه نهایی.

## ویژگی‌ها

### برای دانشجو:
- ثبت نام و ورود به سیستم
- درخواست اخذ پروژه
- پیشنهاد موضوعات پروژه به استاد راهنما
- ارسال گزارش‌های پیشرفت
- مشاهده وضعیت پروژه و زمان دفاعیه
- ارسال پیام به استاد راهنما، داور و مدیر گروه

### برای استاد:
- ثبت نام و ورود به سیستم
- مشاهده دانشجویان تحت سرپرستی (استاد راهنما)
- مشاهده و تایید موضوعات پیشنهادی دانشجویان
- مشاهده گزارش‌های پیشرفت دانشجویان
- مشاهده دانشجویان تحت نظارت (استاد داور)
- پیشنهاد زمان‌های دفاعیه

### برای مدیر گروه:
- تایید درخواست‌های پروژه
- انتخاب استاد راهنما و داور برای هر پروژه
- مشاهده تمام پروژه‌های رشته
- تایید زمان‌های پیشنهادی دفاعیه
- تخصیص خودکار زمان دفاعیه به دانشجویان

## فناوری‌های استفاده شده

- **Backend:** Node.js، Express.js
- **Database:** MongoDB با Mongoose ODM
- **Template Engine:** EJS
- **Session Management:** express-session با connect-mongo
- **Frontend:** HTML، CSS، JavaScript

## نصب و راه‌اندازی

### پیش‌نیازها

- Node.js (نسخه 14 یا بالاتر)
- MongoDB (نسخه 4 یا بالاتر)

### مراحل نصب

1. نصب وابستگی‌ها:
```bash
npm install
```

2. تنظیم متغیرهای محیطی:
فایل `.env` را ویرایش کنید و مقادیر زیر را تنظیم کنید:
```env
PORT=3000
MONGODB_URI=mongodb://localhost:27017/uniProject
SESSION_SECRET=your-secret-key-change-this-in-production
NODE_ENV=development
```

3. اطمینان از اجرای MongoDB:
```bash
# اگر MongoDB به صورت local نصب است:
mongod
```

4. اجرای پروژه:
```bash
# حالت توسعه (با nodemon)
npm run dev

# یا حالت عادی
npm start
```

5. باز کردن مرورگر:
```
http://localhost:3000
```

## راهنمای استفاده

### ثبت نام:

#### دانشجو:
- نام، نام خانوادگی
- شماره دانشجویی (رمز عبور)
- کد ملی (نام کاربری)
- رشته تحصیلی

#### استاد:
- نام، نام خانوادگی
- شماره شناسایی استادی (رمز عبور)
- کد ملی (نام کاربری)
- رشته تحصیلی

#### مدیر گروه:
- نام، نام خانوادگی
- شماره شناسایی مدیرگروه (رمز عبور)
- کد ملی (نام کاربری)
- رشته تحصیلی

### فرآیند کار:

1. **دانشجو درخواست پروژه می‌دهد**
2. **مدیر گروه درخواست را تایید و اساتید را انتخاب می‌کند**
3. **دانشجو موضوعات پیشنهادی می‌فرستد**
4. **استاد راهنما یک موضوع را تایید می‌کند**
5. **دانشجو گزارش‌های پیشرفت ارسال می‌کند**
6. **استاد داور زمان‌های دفاعیه را پیشنهاد می‌دهد**
7. **مدیر گروه زمان‌ها را تایید می‌کند - سیستم خودکار زمان‌بندی می‌کند**
